# 2026-03-18

## KI-Assistent Deployment Issues + Manual Fix

### Problem
GitHub Actions Pipeline für `minicon-assistant-service` scheitert seit gestern (Commit `66f98bc`):
- **Fehler:** `docker: open /opt/config/assistant-service/.env: no such file or directory`
- Später dann: `Permission denied (publickey,password)` bei SSH

### Ursache
1. Ursprünglicher Workflow versuchte `docker run` **lokal** auf dem Runner auszuführen (nicht auf dem Server)
2. Der `.env`-Pfad existiert nur auf dem Server
3. Nach Fix: SSH-Key auf dem Runner wird nicht akzeptiert

### Lösung (Manuell)
- Dateien per SCP auf Server hochgeladen
- Docker-Build und Container-Start manuell auf dem Server ausgeführt
- Alle Features funktionieren jetzt:
  - Registration + E-Mail-Verifikation
  - Login + JWT
  - Chat (Llama 3.1)
  - Kalender/Events CRUD
  - Dokumente Upload/Download
  - Stripe Plans (Free/Basic/Pro)
  - Account Deletion

### Geänderte Files
- `deploy.yml` mehrfach gefixt (Network=web, korrekte URLs, SSH-basiert)
- Letzte Version: `88f0e96` - SSH-basierter Deploy

### Traefik Fix heute Morgen
- Route für `www.minicon.eu` zeigte auf falschen Container (`minicon-eu` → `static-sites-minicon-eu-1`)
- Behoben: Config in `/opt/traefik/config/routes.yml` angepasst

### Automatisches Deployment
- **Status:** Noch nicht vollständig repariert
- **Workaround:** Manuelle Deploys bis SSH-Key auf Runner funktioniert
- Alternativ: Script auf Server erstellen, das per Cron prüft

### Finale Lösung (18.03.2026)
**Pattern erkannt:** TV Dahn nutzt `deploy-static-site.yml` aus `.github` Repo.

**Fix für Assistant-Service:**
1. `deploy.yml` in `minicon-assistant-service` auf Shared Workflow umgestellt:
   ```yaml
   uses: Minicon-eG/.github/.github/workflows/deploy-docker-service.yml@main
   ```
2. Shared Workflow `deploy-docker-service.yml` gefixt: `docker rm` → `docker rm -f`
   (verhindert "container name already in use" Fehler)

**Ergebnis:** Pipeline läuft jetzt erfolgreich! ✅

### Learnings
- **Immer Shared Workflows nutzen** statt eigene defekte Workflows
- **Docker rm -f** erzwingt Entfernung bei Konflikten
- **TV Dahn als Referenz** nutzen (funktioniert seit Wochen)
- Pipeline-Failures: Logs in GitHub Actions → herunterladen und lokal prüfen
